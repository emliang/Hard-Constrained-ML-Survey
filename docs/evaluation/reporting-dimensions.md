
# Reporting Dimensions

Hard-constraint claims should be reported along four axes.

## 1. Constraint satisfaction

Report the guarantee type, residuals, tolerances, violation rates, and recovery failures.

## 2. Output utility

Report task accuracy, objective gap, sample quality, downstream success, diversity, or application-specific utility.

## 3. Training and inference cost

Report training overhead, inference latency, solver or sampler calls, iteration counts, and memory use.

## 4. Scope and assumptions

Report the supported constraint class, domain, solver/oracle/map validity conditions, and discretization assumptions.
