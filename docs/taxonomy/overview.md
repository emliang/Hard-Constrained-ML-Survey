
# Taxonomy Overview

The survey uses a mechanism-first taxonomy. A method is organized by where feasibility is enforced and what supports the feasibility claim.

## Main branches

| Branch | Question |
|---|---|
| Hard-constrained prediction | How does a predictor return feasible decisions or labels? |
| Hard-constrained generation | How does a generator produce valid constrained samples? |
| Applications and benchmarks | How are feasibility, utility, and cost reported in practice? |
| Evaluation terminology | What kind of evidence supports a hard-constraint claim? |

The taxonomy is intentionally not only application-based. The same mechanism can appear in optimization, scientific modeling, structured prediction, constrained decoding, and generative design.
